# 前端交互问题单：小程序C端下拉刷新与上滑加载

更新时间：2026-05-03  
问题级别：P1  
适用范围：前端通用页面交互（非 `/journey` 专项）

## 1. 问题描述

1. 当前部分页面未实现“下拉刷新”，用户无法主动重拉最新后端数据。  
2. 文章列表页缺少“上滑加载更多”，当数据量较大时仅展示首屏数据。  
3. 某些自定义 `scroll-view` 场景下，即使有上滑逻辑，也可能出现“事件未触发”或“触发后被条件拦截”。

## 2. 影响范围

1. 内容中心文章列表（首屏后无更多加载）。  
2. 其他依赖分页数据的业务页（若未统一接入刷新/分页规范）。  
3. 真机调试场景下，滚动触发不稳定更容易暴露问题。

## 3. 建议整改（前端）

1. 所有业务页面统一补齐“下拉刷新”（Pull Down Refresh）。  
2. 文章列表页补齐“上滑加载更多”（Reach Bottom / Infinite Scroll）。  
3. 分页口径统一：  
   1. 首屏：`page=1`  
   2. 上滑：`page++` 并追加列表  
   3. 下拉：重置为 `page=1`，清空旧列表后重拉  
4. 对 `scroll-view` 页面增加触发兜底与调试日志：  
   1. 记录 `hasMore / nextCursor(or page) / loading`  
   2. 校验容器高度、阈值与是否可滚动  
   3. 区分“事件未触发”与“触发后被条件拦截”

## 4. 验收标准

1. 页面下拉后能触发刷新请求并回到最新数据状态。  
2. 文章列表上滑可持续加载下一页，直到 `hasMore=false`。  
3. 真机端滚动触发稳定，无“偶发不触发”或“重复触发导致并发请求”问题。  
4. 分页过程中无重复数据、无跳页、无覆盖已加载数据。
